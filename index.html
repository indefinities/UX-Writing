A Beginner's Guide to User Experience Writing

Introduction

Today, there are constantly new topics appearing within the technology industry, especially within the user experience field. As the field is rapidly changing, it can be difficult to keep up with all the new developments. In order to adjust to the current trends, a useful technique is to use a pen and paper to visualize what is happening. Not only will noting objects and concepts on paper will be beneficial for yourself, it will also be useful for communicating your ideas to others.

As a designer and developer, I believe in creating diagrams in order to fully understand the problem at hand. For example, if a developer is explaining their implementation of a user interface to a designer, then presenting a diagram of their code implementation will help them better communicate implementation choices than to point at certain parts of the code and explain the functionality.

Background

What is a "User Interface (UI) Component"?

User Interface (UI) Components are typically the elements of the user interface (e.a. webpage, application, etc.) that a user interacts with to achieve a certain goal.

(For Developers)

As Javascript is the programming language that connects front-end web development and user experience design, I recommend referencing the Vue framework when reviewing how to create diagrams for user experience components. Vue is a component based Javascript framework where different user interface components can be implemented into their own Vue component file, then those Vue components can be nested into another component which may represent a webpage. The concept of Vue as a framework is relevant to grouping code elements into blocks of a diagram.

Walkthrough

Within a webpage, there are various user interface components that the user interacts with. Let's say we are creating a checkout scenario on an e-commerce website. The three main steps the customer will undergo in this scenario include:

  1. Selecting the payment method
  1. Filling out a form for payment method information
  1. Pressing a button to save information (for future use).

First, selecting a payment method includes selecting a radio button with a label beside it. The radio button should be implemented as its own UI component, as the radio buttons for the different payment methods do not refer to each other.

Following the user selecting the chosen payment method, a form appears. A form may sound like a straightforward component, but it is actually a component made from multiple field components. These "field" components can include, but are not limited to, checkboxes, text inputs, radio boxes, etc. When creating a diagram to represent this information, it is important to group the form field as its own implementation, then reuse the label for that component multiple times to create the hierarchy of the form component.

In this case, it is important to understand the implementation of the field within the form, as there are multiple states the field can return as a result of user input:

  1. The information within the field is correct, and the user can continue to the next step
  1. The field is filled out incorrectly. The user must re-enter information that pertains to the respective field.

Considering these individual implementation is important in order to create the component as a whole, as some components within the whole component will rely on one another for new information.

Conclusion

Understanding the groupings of components and monitoring multiple UI components is useful within software engineering and UX design today. For example, when engineering individual components for the front-end of a website, programmers typically should create a single component and implement that component within multiple areas. However, there is the possibility of miscommunication between programmers, causing a component with similar, if not same, code will be duplicated. Therefore, in this case it's good to document a component's implementation as a diagram and implement that document as a resource. When designing a website interface, it's good to use the "Component" grouping within the program being used (Sketch, Figma, etc.), so the component can be used in multiple places instead of creating the same design repeatedly. Communication and documenting save time and are crucial for product creation.

Using this technique significantly guided me during my time at Chewy and has shaped me to the developer and designer I am today. With the tactic of noting implementation of work using pen and paper, visual learners will have the potential to learn anything out of their comfort zone.
